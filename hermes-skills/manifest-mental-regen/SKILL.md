# Manifest Mental Regeneration

## Overview

Regeneration is not revision. If the user rejects a post, the replacement must learn from the rejection and create a stronger new candidate. It should not preserve the same weak sentence with small edits.

## When To Use

Use this skill when:

- The user rejects one or more posts.
- The user gives taste feedback.
- The replacement must explain what it learned.

## Regeneration Procedure

1. Identify what the user rejected.
2. Separate fixable signal from banned direction.
3. Decide whether to preserve the core premise or abandon it.
4. Create a new post with a new sentence path.
5. Explain briefly what was learned.
6. Write a durable memory note.

## Preserve vs Abandon

Preserve when:

- The user says “this could be strong if...”
- The issue is one word, ending, or sentence shape.
- The premise is strong but the payoff is weak.

Abandon when:

- The user says it is AI slop.
- The sentence makes no sense.
- The image is vague or corny.
- The core metaphor is the problem.

## Hard Rules

- Do not return the same post with synonyms.
- Do not keep the same main nouns and ending logic.
- Do not over-explain in the replacement.
- Do not use the user’s feedback as a prompt to become more generic.
- If the user gives a better wording direction, learn from the principle, not just the exact phrase.

## Verification Checklist

- [ ] The replacement is materially different.
- [ ] The user’s reason is addressed.
- [ ] The memory note captures a reusable taste rule.
- [ ] The new line has a clearer payoff than the rejected line.
