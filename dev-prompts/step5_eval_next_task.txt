EVAL / NEXT TASK

Status
This legacy VS Code handoff prompt is deprecated.

Purpose
If this file is still used anywhere, keep the interaction lightweight and do not
depend on `CODEX RESPONSE`, `Run These Now`, or any fixed response wrapper.

Rules
- Accept plain-language summaries from the coding agent.
- Do not require a response header.
- Do not require `Summary`, `Files Modified`, `Run These Now`, or
  `Risks / Edge Cases` sections.
- Do not mirror tasks from assistant output.
- Ask only for genuinely missing verification or deploy status.
- Prefer a single concise follow-up over rigid step orchestration.

Default Behavior
- If the latest assistant message clearly states what changed and what was or
  was not verified, accept it.
- If a required check or deploy result is actually missing, ask for that result
  in plain language.
- If a task failed, request a correction in plain language without imposing a
  template.
